1. ¿Qué es el Model Context Protocol (MCP)?

Antes de construir, necesitamos entender la tecnología central de este tutorial.

🔌 Definición de MCP

Model Context Protocol (MCP) es un protocolo abierto (creado por Anthropic) que permite a los modelos de IA conectarse con herramientas y datos externos de forma estandarizada. Es como un "USB universal" para que la IA use herramientas del mundo real.

1.1 Analogía: El Traductor Universal

❌ Sin MCP

Le pides a Claude: "Analiza esta factura"

Claude solo puede leer texto que le pegues manualmente. No puede acceder a archivos, bases de datos ni enviar mensajes.

✅ Con MCP

Le pides a Claude: "Analiza esta factura"

Claude detecta automáticamente que tiene una herramienta "analizar_factura", la ejecuta en n8n, recibe el resultado y te lo muestra.

1.2 Arquitectura Cliente-Servidor

Componente Rol En Nuestro Caso
MCP Client Modelo de IA que quiere usar herramientas Claude Desktop — donde escribes en lenguaje natural
MCP Server Servidor que expone herramientas al cliente n8n — expone workflows como "tools"
Tools Herramientas específicas que el servidor ofrece 4 workflows: analizar, guardar, notificar, consultar
Transporte Protocolo de comunicación entre cliente y servidor SSE (Server-Sent Events) sobre HTTP

1.3 ¿Por qué n8n como MCP Server?

🤖 El Poder de la Combinación

n8n tiene más de 400 integraciones (Notion, Telegram, Google, Slack, bases de datos, APIs...). Al exponerlas como MCP Server, Claude puede acceder a todas estas herramientas simplemente con una instrucción en lenguaje natural. No necesitas programar cada interacción — lo diseñas visualmente en n8n y Claude lo ejecuta.

💡 La Revolución del Paradigma

Antes: Tú construyes un flujo → configuras un webhook → envías datos con Postman → ves el resultado.
Ahora: Tú le dices a Claude "analiza la factura de X proveedor y guárdala en Notion" → Claude ejecuta automáticamente los workflows necesarios → te muestra el resultado en una conversación.

2. Workflow Principal — MCP Server (Orquestador)

Este workflow es el punto de entrada. Expone los 4 tools a Claude Desktop.

2.1 Crear el Workflow MCP Server

1Crear nuevo workflow

En n8n (http://localhost:5678), haz clic en "Create Workflow".

Renómbralo a: MCP Server - Facturas Inteligentes

2Agregar nodo MCP Server Trigger

  1. Haz clic en el botón "+" del canvas
  2. Busca "MCP Server Trigger"
  3. Selecciona el nodo "MCP Server Trigger"

3Configurar el MCP Server Trigger

En el panel de configuración:

  • Path: Deja el path generado automáticamente o pon facturas-mcp
  • Authentication: Bearer Auth

Para la autenticación Bearer:

  1. Selecciona "Bearer Auth" en Authentication
  2. Haz clic en "Create New Credential"
  3. En el campo Token, escribe un token seguro, por ejemplo: mi-token-mcp-facturas-2026
  4. Guarda la credencial

Anota dos valores importantes:

  • La Production URL del MCP (se ve en la parte superior del nodo, ej: http://localhost:5678/mcp/facturas-mcp/sse)
  • El Token Bearer que configuraste

⚠️ Importante: Usa la URL de Producción

Claude Desktop se conectará a la Production URL, no a la Test URL. Esto significa que el workflow debe estar activado (publicado) para que funcione.

2.2 Agregar los 4 Custom Workflow Tools

Ahora conectaremos 4 nodos "Custom n8n Workflow Tool" al MCP Server Trigger. Cada uno representa un tool que Claude podrá usar.

4Tool 1 — Analizar Factura

  1. Haz clic en el "+" del MCP Server Trigger
  2. Busca "Custom n8n Workflow Tool" (o "Call n8n Workflow Tool")
  3. Configura:
  • Name: analizar_factura
  • Description: Analiza una factura desde una URL de PDF. Extrae proveedor, número de factura, fecha, items, subtotal, impuestos y total. Recibe como parámetro la URL del archivo PDF de la factura.
  • Workflow: Selecciona el workflow "Tool - Analizar Factura" (lo crearemos después)

5Tool 2 — Guardar en Notion

  1. Agrega otro "Custom n8n Workflow Tool"
  2. Configura:
  • Name: guardar_factura_notion
  • Description: Guarda los datos de una factura analizada en la base de datos de Notion. Recibe: proveedor, numero_factura, fecha, subtotal, impuestos, total, moneda, items, estado.
  • Workflow: Selecciona "Tool - Guardar en Notion"

6Tool 3 — Notificar por Telegram

  1. Agrega otro "Custom n8n Workflow Tool"
  2. Configura:
  • Name: notificar_telegram
  • Description: Envía un mensaje de notificación por Telegram con el resumen de una factura o cualquier mensaje relevante. Recibe: mensaje (texto a enviar).
  • Workflow: Selecciona "Tool - Notificar Telegram"

7Tool 4 — Consultar Facturas

  1. Agrega otro "Custom n8n Workflow Tool"
  2. Configura:
  • Name: consultar_facturas
  • Description: Consulta las facturas almacenadas en Notion. Puede filtrar por proveedor. Recibe: proveedor (opcional, nombre del proveedor para filtrar).
  • Workflow: Selecciona "Tool - Consultar Facturas"

💡 La Descripción es CLAVE

Claude decide qué tool usar basándose exclusivamente en la descripción. Si escribes una descripción vaga, Claude no sabrá cuándo usarlo. Sé específico sobre qué hace el tool y qué parámetros recibe.

3. Tool 1 — Analizar Factura (5 nodos)

Este workflow descarga un PDF de factura, extrae el texto y usa Claude para analizarlo.

3.1 Crear el Workflow

1Nuevo workflow

Crea un nuevo workflow llamado: Tool - Analizar Factura

3.2 Nodo 1 — Execute Workflow Trigger

2Agregar Execute Workflow Trigger

  1. Busca "Execute Workflow Trigger"
  2. Este nodo es el punto de entrada cuando el MCP Server llama a este workflow
  3. No requiere configuración especial — recibe automáticamente los parámetros que Claude envía

3.3 Nodo 2 — HTTP Request (Descargar PDF)

3Agregar HTTP Request

Configuración:

  • Method: GET
  • URL: ={{ $json.query }} (la URL viene del parámetro que envía Claude)
  • Response Format: File
  • Output File Name: factura.pdf

💡 ¿De dónde viene $json.query?

Cuando Claude llama a un tool, envía los parámetros como texto en el campo query. El Custom Workflow Tool de n8n pasa este valor al sub-workflow. En este caso, Claude enviará la URL del PDF.

3.4 Nodo 3 — Extract From File

4Agregar Extract From File

Configuración:

  • Operation: Extract From PDF
  • Input Binary Field: data (por defecto)

Este nodo extrae el texto del PDF descargado. El resultado estará en $json.text.

3.5 Nodo 4 — Anthropic (Analizar con IA)

5Agregar nodo Anthropic (Message a Model)

Busca "Anthropic" y selecciona la acción "Message a Model".

Primero, configura la credencial:

  1. Haz clic en "Create New Credential"
  2. Pega tu API Key de Anthropic (sk-ant-...)
  3. Guarda

Configuración del nodo:

  • Model: claude-sonnet-4-20250514
  • System Prompt (Messages → System):
Eres un experto en análisis de facturas y documentos contables. 
Tu tarea es extraer datos estructurados de facturas.

SIEMPRE responde en formato JSON válido con esta estructura exacta:
{
  "proveedor": "nombre del proveedor o empresa emisora",
  "numero_factura": "número o código de la factura",
  "fecha": "YYYY-MM-DD",
  "items": [
    {"descripcion": "...", "cantidad": 0, "precio_unitario": 0, "subtotal": 0}
  ],
  "subtotal": 0,
  "impuestos": 0,
  "total": 0,
  "moneda": "COP o USD o EUR",
  "notas": "cualquier observación relevante"
}

Si algún campo no está claro en el documento, usa "No especificado" para texto o 0 para números.
Responde SOLO con el JSON, sin texto adicional.
  • User Message:
Analiza el siguiente texto extraído de una factura y extrae todos los datos estructurados:

{{ $json.text }}
  • Max Tokens: 2000

3.6 Nodo 5 — Edit Fields (Estructurar Respuesta)

6Agregar Edit Fields

Este nodo formatea la salida para que el MCP Server la devuelva a Claude correctamente.

  • Mode: Manual Mapping
  • Fields:
Name Value
response ={{ $json.message.content[0].text }}

Esto toma la respuesta de Claude (el JSON con datos de la factura) y la pone en un campo limpio.

4.3 Nodo 2 — Code (Parsear datos)

3Agregar nodo Code

Claude enviará los datos como texto. Necesitamos parsearlo a JSON:

  • Language: JavaScript
  • Code:
// Parsear los datos de la factura que envía Claude
const input = $input.first().json.query;
let data;

try {
  data = JSON.parse(input);
} catch (e) {
  // Si no es JSON válido, intentar extraer datos del texto
  data = {
    proveedor: input,
    numero_factura: "No especificado",
    fecha: new Date().toISOString().split('T')[0],
    subtotal: 0,
    impuestos: 0,
    total: 0,
    moneda: "COP",
    items: "No especificado",
    estado: "Pendiente"
  };
}

return [{
  json: {
    proveedor: data.proveedor || "No especificado",
    numero_factura: data.numero_factura || "No especificado",
    fecha: data.fecha || new Date().toISOString().split('T')[0],
    subtotal: parseFloat(data.subtotal) || 0,
    impuestos: parseFloat(data.impuestos) || 0,
    total: parseFloat(data.total) || 0,
    moneda: data.moneda || "COP",
    items: typeof data.items === 'object' 
      ? JSON.stringify(data.items) 
      : String(data.items || "No especificado"),
    estado: data.estado || "Pendiente"
  }
}];

6.5 Nodo 4 — Code (Formatear Resultados)

5Agregar nodo Code

Este nodo formatea la respuesta de Notion para que sea legible:

  • Language: JavaScript
  • Code:
// Formatear las facturas de Notion para Claude
const items = $input.all();
const facturas = items.map(item => {
  const props = item.json.properties;
  return {
    proveedor: props.Proveedor?.title?.[0]?.plain_text || "N/A",
    numero: props["Numero Factura"]?.rich_text?.[0]?.plain_text || "N/A",
    fecha: props["Fecha Factura"]?.date?.start || "N/A",
    subtotal: props.Subtotal?.number || 0,
    impuestos: props.Impuestos?.number || 0,
    total: props.Total?.number || 0,
    moneda: props.Moneda?.select?.name || "COP",
    estado: props.Estado?.select?.name || "N/A"
  };
});

const resumen = {
  total_facturas: facturas.length,
  suma_total: facturas.reduce((acc, f) => acc + f.total, 0),
  facturas: facturas
};

return [{
  json: {
    response: JSON.stringify(resumen, null, 2)
  }
}];

7. Conectar Claude Desktop a n8n (MCP)

Este es el paso mágico: conectar Claude Desktop con tu servidor MCP en n8n.

7.1 Localizar el Archivo de Configuración

1Abrir el archivo de configuración de Claude Desktop

En Windows, el archivo está en:

%APPDATA%\Claude\claude_desktop_config.json

Para abrirlo rápidamente, en PowerShell ejecuta:

notepad "$env:APPDATA\Claude\claude_desktop_config.json"

Si el archivo no existe, créalo.

7.2 Configurar la Conexión MCP

2Pegar la configuración

Reemplaza el contenido del archivo con:

{
  "mcpServers": {
    "n8n-facturas": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "http://localhost:5678/mcp/<TU_PATH>/sse",
        "--header",
        "Authorization: Bearer ${AUTH_TOKEN}"
      ],
      "env": {
        "AUTH_TOKEN": "mi-token-mcp-facturas-2026"
      }
    }
  }
}

Reemplaza:

  • <TU_PATH> → El path que configuraste en el MCP Server Trigger (ej: facturas-mcp o el path autogenerado)
  • mi-token-mcp-facturas-2026 → El token Bearer que configuraste en la credencial

⚠️ Obtener la URL exacta

La forma más segura: abre el nodo MCP Server Trigger en n8n → cambia a "Production URL" → copia la URL completa. Debería verse como: http://localhost:5678/mcp/abc123/sse

9. Troubleshooting — Solución de Problemas

9.1 Errores de Conexión MCP

Problema Causa Probable Solución
Claude no muestra los tools Configuración incorrecta Verifica el archivo claude_desktop_config.json: la URL y el token deben ser correctos. Reinicia Claude Desktop.
"Connection refused" en Claude n8n no está ejecutándose Verifica que n8n esté corriendo en localhost:5678. El workflow MCP Server debe estar activo.
"401 Unauthorized" Token Bearer incorrecto El token en claude_desktop_config.json debe coincidir exactamente con el de la credencial en n8n.
"npx: command not found" Node.js no instalado Instala Node.js desde nodejs.org. Reinicia tu computadora después de instalar.
Tools aparecen pero fallan Sub-workflows no guardados Verifica que los 4 sub-workflows estén guardados y seleccionados correctamente en los Custom Workflow Tools.
"Timeout" al ejecutar tool Workflow tarda demasiado El PDF podría ser muy grande. Prueba con un PDF más pequeño o aumenta el timeout en n8n.

9.2 Errores de Servicios

Problema Causa Probable Solución
Notion: "Could not find database" Integration no conectada En Notion, ve a la base de datos → ⋯ → Connections → agrega tu integration "n8n-facturas".
Telegram: "Chat not found" Chat ID incorrecto Verifica tu Chat ID en api.telegram.org/bot<TOKEN>/getUpdates. Envía primero un mensaje al bot.
Anthropic: "insufficient_quota" Sin créditos Ve a console.anthropic.com → Billing y agrega créditos.
HTTP Request: error al descargar PDF URL inaccesible Verifica que la URL del PDF sea pública y accesible. Prueba abrirla directamente en el navegador.
Extract From File: "Unsupported format" Archivo no es PDF Asegúrate de que la URL apunta a un archivo PDF real, no a una página web.

9.3 Tips de Depuración

🔍 Inspeccionar Ejecuciones en n8n

Ve a Executions en n8n para ver exactamente qué datos recibió cada nodo y dónde falló la cadena.

🧪 Probar Sub-workflows Independientemente

Puedes ejecutar cada sub-workflow manualmente con datos de prueba (Pin Data) antes de conectarlo al MCP Server.

📋 Revisar Logs de Claude Desktop

En Windows: %APPDATA%\Claude\logs\. Los logs muestran errores de conexión MCP detallados.

💡 Probar la URL MCP en el Navegador

Abre la URL del MCP en el navegador. Si ves un error 401, la autenticación funciona. Si ves "connection refused", n8n no está corriendo.

10. Reto Final — Extender el Sistema

Ahora que tienes el sistema funcionando, extiéndelo con estas ideas:

🏆 Reto 1: Alertas de Vencimiento
  • Agrega un campo "Fecha de Vencimiento" a la base de Notion
  • Crea un nuevo Tool que consulte facturas próximas a vencer
  • Pídele a Claude: "¿Tengo facturas por vencer esta semana?"
🏆 Reto 2: Análisis de Gastos
  • Crea un Tool que agrupe facturas por proveedor y calcule totales
  • Pídele a Claude: "Dame un resumen de gastos por proveedor del último mes"
  • Incluye gráficos con QuickChart (nodo HTTP Request)
🏆 Reto 3: Multi-documento
  • Modifica el Tool de análisis para que soporte otros documentos (cotizaciones, órdenes de compra)
  • Usa el Switch node para enrutar según el tipo de documento detectado por la IA
  • Guarda cada tipo en una base de datos diferente de Notion
🧠 Reto 4: Detección de Anomalías
  • Crea un Tool que compare el monto de una factura nueva con el historial
  • Si el monto es significativamente mayor al promedio, alerta por Telegram
  • Pídele a Claude: "¿Esta factura parece normal comparada con las anteriores?"
