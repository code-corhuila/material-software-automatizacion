1. Entender la Arquitectura Avanzada

Antes de codificar, comprendamos los componentes del sistema.

1.1 Estructura de Datos que Recibiremos

El webhook recibirá datos POST con esta estructura:

{
  "nombre": "Juan García López",
  "email": "juan.garcia@example.com",
  "empresa": "Tech Solutions SAS",
  "telefono": "+57 3001234567",
  "asunto": "Consulta sobre automatización",
  "mensaje": "Quisiera saber cómo implementar n8n en nuestra empresa",
  "prioridad": "alta",
  "origen": "formulario-web"
}

1.2 Validaciones que Implementaremos

Email

Validar formato de email usando regex: usuario@dominio.com

Nombre

Verificar que no esté vacío y tenga al menos 3 caracteres.

Mensaje

Comprobar que el mensaje no sea más de 5000 caracteres.

Teléfono

Validar que sea un número válido (opcional pero verificado).

1.3 Enriquecimiento de Datos

Antes de enviar el email, agregaremos automáticamente:

  • Timestamp: Fecha y hora exacta de la solicitud
  • ID Único: Un identificador único para rastrear la solicitud
  • IP Address: De dónde vino la solicitud (si disponible)
  • Estado: "NUEVO" para nuevas solicitudes
  • Asignado a: Equipo responsable (ej: "Soporte Técnico")

💡 ¿Por qué Enriquecer Datos?

El enriquecimiento de datos es una práctica común en automatización. Agregar metadatos (datos sobre los datos) nos permite:

  • Rastrear el origen y auditoría de cada solicitud
  • Identificar patrones o problemas
  • Automatizar el direccionamiento a equipos correctos
  • Mantener un histórico completo en Google Sheets

2. Crear Flujo Avanzado Paso a Paso

2.1 Crear Nuevo Flujo

1Acceder a n8n

Abre tu navegador en http://localhost:5678 e inicia sesión.

2Crear nuevo flujo

Haz clic en "Create Workflow" o ve a Dashboard → "+ New Workflow".

3Nombrar el flujo

Haz clic en el título "Untitled workflow" y renómbralo a:

Sistema Avanzado de Contacto - Formulario Web

Presiona Enter.

2.2 Agregar Nodo Webhook

4Buscar y agregar Webhook

En el panel izquierdo, busca webhook.

Haz clic en "Webhook" para agregarlo al canvas.

5Configurar Webhook

Haz doble clic en el nodo Webhook. Configura:

  • HTTP Method: POST
  • Path: contacto-formulario (sin barra inicial)
  • Authentication: None

Copia la URL completa del webhook (la verás en el nodo), que será algo como:

http://localhost:5678/webhook/contacto-formulario

Guardaremos esta URL para pruebas más adelante.

2.3 Nodo de Validación Avanzada (IF)

6Agregar primer nodo IF

Busca if y haz clic en "IF".

Conecta el Webhook con el IF (arrastra desde punto de salida del Webhook).

7Configurar validación de Email

Haz doble clic en el nodo IF. Configura la primera condición:

  • Campo a validar: {{ $json.email }}
  • Operador: "matches regex"
  • Patrón regex:
^[a-zA-Z0-9_%.+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]{2,}$

Esto valida que el email tenga formato correcto. El nodo IF tendrá dos ramas:

  • TRUE (rama izquierda): Email válido, continúa
  • FALSE (rama derecha): Email inválido, error

💡 Consejo: IF Node

En n8n, el nodo IF crea dos ramas de ejecución. En este caso, solo continuaremos por la rama TRUE. Podríamos agregar manejo de errores en la rama FALSE más adelante, pero por ahora nos enfocaremos en el flujo correcto.

2.4 Nodo para Enriquecer Datos

8Agregar nodo Set (para enriquecer datos)

Busca set y haz clic en "Set" (o "Assign").

Conecta desde la rama TRUE del IF al nodo Set.

9Configurar el enriquecimiento

Haz doble clic en el nodo Set. Aquí agregaremos campos adicionales a los datos:

Pasos para agregar campos:
  1. Busca el botón "Add Field" o similar
  2. Para cada campo nuevo, haz clic en "Add Field"
Agrega estos campos:

1. Timestamp

  • Nombre del campo: timestamp
  • Valor: {{ $now.toISO() }}

2. ID Único (UUID)

  • Nombre del campo: id_solicitud
  • Valor: {{ $randomUUID() }}

3. Estado

  • Nombre del campo: estado
  • Valor: "NUEVO" (incluye las comillas)

4. Equipo Responsable

  • Nombre del campo: asignado_a
  • Valor: "Soporte Técnico"

5. Origen Sistema

  • Nombre del campo: origen_sistema
  • Valor: "Formulario Web - Sistema n8n"

💡 Entender expresiones n8n

  • {{ }} = Delimitadores para expresiones
  • $now.toISO() = Fecha/hora actual en formato ISO
  • $randomUUID() = Genera un ID único
  • "texto" = Cadena de texto (requiere comillas)

2.5 Nodo Gmail para Notificación

10Agregar nodo Gmail

Busca gmail y haz clic en "Gmail".

Conecta desde el nodo Set al Gmail.

11Configurar Gmail

Haz doble clic en Gmail. Si ya tienes OAuth configurado del tutorial anterior, aparecerá tu email.

Configuración básica:
  • Gmail Account: Tu correo (debe estar autenticado)
  • To: Tu dirección de email o {{ $json.email }} para enviarle al remitente
  • Subject: Nueva Solicitud de Contacto - {{ $json.nombre }}

✅ Progreso Hasta Aquí

Flujo: Webhook → IF (validar email) → Set (enriquecer) → Gmail (notificar)

Ya tenemos un flujo funcional. Ahora agregaremos Google Sheets para auditoría.

3. Google Sheets para Auditoría y Registro

Guardaremos cada solicitud en Google Sheets para tener un histórico completo.

3.1 Crear Google Sheet

1Crear nuevo Google Sheet

Antes de agregar el nodo a n8n, crea la hoja en Google Sheets:

  1. Abre sheets.google.com
  2. Crea un nuevo Sheet (documento en blanco)
  3. Nómbralo: Contactos - Sistema n8n
  4. En la primera fila, crea estas columnas:
ID | Nombre | Email | Empresa | Teléfono | Asunto | Mensaje | Prioridad | Fecha/Hora | Estado | Asignado a

3.2 Agregar Nodo Google Sheets

2Agregar nodo Google Sheets

De vuelta en n8n, busca google sheets y haz clic en "Google Sheets".

Conecta desde el nodo Set (o desde Gmail si prefieres) al Google Sheets.

3Autenticar Google Sheets

Haz doble clic en el nodo Google Sheets. Ve al campo "Credential".

Haz clic en "Create Credentials" o en el dropdown de credenciales.

Se abrirá una ventana pidiendo seleccionar tu cuenta Google. Elige la misma que usaste para Gmail.

Haz clic en "Allow" cuando solicite permisos.

4Configurar Google Sheets

Después de autenticar:

  • Operation: Selecciona "Append" (agregar filas)
  • Spreadsheet: Busca y selecciona "Contactos - Sistema n8n" que creaste
  • Sheet: Selecciona "Sheet1" o la que corresponda

5Mapear Columnas

En la sección de datos a insertar, mapea cada columna:

  • ID: {{ $json.id_solicitud }}
  • Nombre: {{ $json.nombre }}
  • Email: {{ $json.email }}
  • Empresa: {{ $json.empresa }}
  • Teléfono: {{ $json.telefono }}
  • Asunto: {{ $json.asunto }}
  • Mensaje: {{ $json.mensaje }}
  • Prioridad: {{ $json.prioridad }}
  • Fecha/Hora: {{ $json.timestamp }}
  • Estado: {{ $json.estado }}
  • Asignado a: {{ $json.asignado_a }}

✅ Flujo Completamente Funcional

Ahora tienes un flujo completo:

Webhook → IF (validar) → Set (enriquecer) → Gmail + Google Sheets

Todas las solicitudes se registran automáticamente en la hoja de cálculo.

4. Variables Avanzadas y Técnicas

4.1 Expresiones Comunes en n8n

Expresión Descripción Ejemplo
${{ $json.campo }} Acceder a un campo de los datos JSON ${{ $json.nombre }} → "Juan"
${{ $now.toISO() }} Fecha/hora actual en formato ISO 2026-02-26T10:30:45.000Z
${{ $randomUUID() }} Generar ID único a1b2c3d4-e5f6-7g8h-9i0j-k1l2m3n4o5p6
${{ $json.texto.toUpperCase() }} Convertir texto a mayúsculas "hola" → "HOLA"
${{ $json.texto.toLowerCase() }} Convertir texto a minúsculas "HOLA" → "hola"
${{ $json.numero * 2 }} Operaciones matemáticas 5 * 2 = 10
${{ $now.toFormat('dd/MM/yyyy') }} Formato personalizado de fecha 26/02/2026

4.2 Validaciones Avanzadas Regex

Email
^[a-zA-Z0-9_%.+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]{2,}$
Teléfono (formato +57)
^\+57\s?\d{10}$
URL
^https?:\/\/(www\.)?[-a-zA-Z0-9@:%._\+~#=]{1,256}\.[a-zA-Z0-9()]{1,6}\b([-a-zA-Z0-9()@:%_\+.~#?&/=]*)$
Solo números (4-6 dígitos)
^\d{4,6}$

6. Solución de Problemas Comunes

Problema 1: El webhook no recibe datos

⚠️ Problema

En Postman enviaste la solicitud pero n8n no registró nada.

Soluciones

  1. Verifica que el flujo está activado: El toggle debe estar verde/on
  2. Copia la URL correcta: Asegúrate de copiar completa desde el nodo Webhook
  3. Usa POST, no GET: El header debe decir "POST" no "GET"
  4. Revisa el Content-Type: Debe ser "application/json"
  5. Verifica n8n está corriendo: http://localhost:5678 debe estar accesible

Problema 2: Email no se envía

⚠️ Problema

El flujo ejecuta pero no recibiste el email.

Soluciones

  1. Revisa credenciales Gmail: Verifica que esté autenticado correctamente
  2. Verifica el campo "To": Debe tener un email válido
  3. Revisa carpeta Spam: A veces los emails llegan ahí
  4. Verifica la ejecución: En "Executions" verás si hay error en el nodo Gmail
  5. Re-autentica Gmail: Borra las credenciales y vuelve a hacer "Create Credentials"

Problema 3: Google Sheets no se actualiza

⚠️ Problema

El flujo ejecuta pero no aparecen datos en Google Sheets.

Soluciones

  1. Verifica credenciales Google: Asegúrate que está autenticado
  2. Nombre correcto del Sheet: El spreadsheet debe existir con el nombre exacto
  3. Permisos de Google Sheets API: Habilita la API en Google Cloud Console
  4. Revisa los mapeos: Los campos deben corresponder con las columnas de la hoja
  5. Recarga la hoja: A veces necesitas recargar el navegador para ver cambios

Problema 4: Validación IF no funciona

⚠️ Problema

El IF no valida correctamente o siempre pasa la rama true/false.

Soluciones

  1. Revisa la regex: Copia exactamente: ^[a-zA-Z0-9_%.+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]{2,}$
  2. Verifica el campo: Asegúrate que escribiste {{ $json.email }}
  3. Prueba la regex online: Usa regex101.com para validar
  4. Revisa los datos enviados: En Postman, asegúrate que el email está en el JSON

Problema 5: El flujo corre muy lentamente

⚠️ Problema

Los nodos se ejecutan con retrasos notables.

Soluciones

  1. Reinicia Docker: Abre Docker Desktop y reinicia los contenedores
  2. Verifica recursos: Asegúrate que tu máquina tiene suficiente RAM/CPU
  3. Revisa logs de Docker: docker-compose logs -f en la terminal
  4. Recarga n8n: Cierra la pestaña del navegador y abre nuevamente

Problema 6: Expresiones no funcionan ({{ }} no se evalúan)

⚠️ Problema

Ves {{ $json.nombre }} literalmente en el email en lugar del nombre real.

Soluciones

  1. En nodos con HTML: El campo debe estar configurado para aceptar expresiones
  2. Verifica modo de edición: En Gmail, asegúrate que el Body está en modo "Text with Expressions"
  3. Usa siempre {{ }}: Toda expresión debe estar entre llaves dobles
  4. Revisa la sintaxis: $json.campo es correcto. $json[campo] sin comillas también puede funcionar
